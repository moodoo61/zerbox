# تحديث نظام البث المباشر - التوثيق

## 📋 ملخص التعديلات

تم تحديث نظام البث المباشر لإزالة التفعيل اليدوي واستبداله بنظام تفعيل تلقائي يعتمد على ملف مفتاح محلي.

---

## 🔄 التغييرات الرئيسية

### 1️⃣ **قراءة المفتاح المحلي**
- يتم البحث عن ملف المفتاح في المسارات التالية:
  - `key.json`
  - `kay.json` (الملف الموجود حالياً)
  - `/root/Zero/key.json`
  - `/root/Zero/kay.json`

### 2️⃣ **التحقق من المفتاح وجلب القنوات**
- يتم إرسال المفتاح إلى: `https://to.zerolag.live/api/channels/verify-key/`
- يتم جلب قائمة القنوات من الخادم الخارجي
- الشكل المتوقع للبيانات:
```json
{
  "channels": {
    "القناة_1": {
      "الرابط": "dtsc://example.com/stream1",
      "المشغل": "player_type",
      "ملاحضه": "ملاحظات حول القناة"
    },
    "القناة_2": {
      "الرابط": "dtsc://example.com/stream2",
      "ملاحضه": "قناة رياضية"
    }
  }
}
```

### 3️⃣ **حذف القنوات القديمة**
- يتم حذف جميع القنوات القديمة من MistServer
- يتم حذف جميع القنوات القديمة من قاعدة البيانات

### 4️⃣ **إضافة القنوات الجديدة**
- يتم إضافة القنوات الجديدة إلى MistServer
- يتم إضافة القنوات الجديدة إلى قاعدة البيانات
- يتم تجاهل حقل "المشغل" كما طُلب

---

## 🛠️ الملفات المعدلة

### Backend

#### `/root/Zero/backend/services.py`
**الدوال الجديدة:**
1. `read_local_key()` - قراءة المفتاح من الملف المحلي
2. `verify_key_and_fetch_channels(key)` - التحقق من المفتاح وجلب القنوات
3. `delete_all_mistserver_streams()` - حذف جميع القنوات من MistServer

**الدوال المعدلة:**
1. `activate_streaming_service(db, subscription_data=None)` - تم تحديثها للتفعيل التلقائي

#### `/root/Zero/backend/main.py`
**نقاط النهاية المعدلة:**
- `POST /api/streaming/activate` - لم تعد تحتاج إلى `subscription_data` في الـ request body

### Frontend

#### `/root/Zero/frontend/src/components/StreamingManager.js`
**التعديلات:**
1. إزالة حقل إدخال `subscription_data` من الواجهة
2. تحديث دالة `handleSubscriptionSubmit` لإرسال طلب بدون بيانات يدوية
3. إضافة رسالة توضيحية عن آلية التفعيل التلقائي

## 🚀 كيفية الاستخدام

### 1. تحضير ملف المفتاح
قم بإنشاء أو تحديث ملف `key.json` في المسار `/root/Zero/`:
```json
{
  "7979385421a5e88619bfc8fae21759c447bff68da7ac24389397d1a31040069d": "1"
}
```

### 2. تشغيل الخدمة
```bash
cd /root/Zero
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8080 --reload
```

### 3. التفعيل من الواجهة
1. افتح لوحة التحكم
2. اذهب إلى "إدارة البث المباشر"
3. اضغط على "تفعيل الخدمة"
4. سيتم التفعيل تلقائياً باستخدام المفتاح من الملف

### 4. التحقق من النجاح
- سيتم عرض رسالة نجاح
- سيتم عرض قائمة القنوات المضافة
- سيتم تحديث حالة الخدمة إلى "مفعل"


## ⚠️ معالجة الأخطاء

### خطأ: "لم يتم العثور على ملف المفتاح"
**الحل:** تأكد من وجود ملف `key.json` أو `kay.json` في المسار الصحيح

### خطأ: "فشل التحقق من المفتاح"
**الحل:** 
- تحقق من اتصال الإنترنت
- تأكد من صحة المفتاح
- تحقق من أن الخادم الخارجي يعمل

### خطأ: "فشل في حذف القنوات"
**الحل:**
- تحقق من أن MistServer يعمل
- تحقق من صحة بيانات الاتصال بـ MistServer

### خطأ: "فشل في إضافة القنوات"
**الحل:**
- تحقق من صحة روابط القنوات
- تحقق من أن MistServer يدعم نوع البروتوكول المستخدم

## 📝 ملاحظات مهمة

1. **المفتاح المحلي:** يجب أن يكون ملف المفتاح موجوداً قبل التفعيل
2. **حقل "المشغل":** يتم تجاهله كما طُلب ولا يتم استخدامه
3. **حذف القنوات:** يتم حذف جميع القنوات القديمة قبل إضافة الجديدة
4. **الترتيب:** يتم الحفاظ على ترتيب القنوات كما يأتي من الخادم
5. **الملاحظات:** يتم حفظ حقل "ملاحضه" في حقل category

---

## 🔐 الأمان

- المفتاح يُقرأ من ملف محلي ولا يُخزن في قاعدة البيانات بشكل كامل
- يتم عرض جزء مختصر من المفتاح فقط (أول 20 حرف)
- يتم إرسال المفتاح عبر HTTPS إلى الخادم الخارجي
