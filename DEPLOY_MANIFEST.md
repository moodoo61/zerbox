# قائمة النشر — المشاريع والخدمات المرتبطة بـ Zero

هذا الملف يوضح **كل المشاريع والخدمات** المرتبطة بمشروع Zero حتى تعرف ماذا تنقل أو تعيد تثبيته عند النشر على جهاز آخر.

---

## 1. المشروع الرئيسي (داخل مجلد Zero)

| المكون | المسار | إعادة البناء على جهاز جديد |
|--------|--------|----------------------------|
| **Backend** | `backend/` | `pip install -r backend/requirements.txt` |
| **Frontend** | `frontend/` | `cd frontend && npm install && npm run build` |
| **قاعدة البيانات** | `database.db` (جذر المشروع) | انسخ الملف مع المشروع |
| **الملفات المرفوعة** | `backend/static/images/` أو `uploads/` | انسخ المجلد مع المشروع |
| **مفتاح التفعيل** | `key.json` أو `kay.json` في جذر المشروع | انسخ الملف مع المشروع |

---

## 2. مشاريع مرتبطة **داخل** مجلد Zero

هذه المجلدات جزء من شجرة Zero؛ عند نسخ مجلد Zero تُنسخ معه، لكن **يجب إعادة تثبيت الاعتماديات وبناءها** على الجهاز الجديد.

| المشروع | المسار | الاستخدام | إعادة البناء على جهاز جديد |
|---------|--------|-----------|----------------------------|
| **القرآن الكريم** | `quran/` | خدمة افتراضية من لوحة التحكم | `cd quran && npm install && npm run build` (أو `npm run serve` حسب السكربت) |
| **قافية** | `qafiyah/` | خدمة افتراضية (منصة الشعر) | `cd qafiyah && npm install` ثم `cd apps/web && npm install` (أو حسب بنية المشروع) |

---

## 3. خدمات نظام (خارج مجلد Zero)

هذه **لا تُنسخ** مع مجلد المشروع؛ يجب تثبيتها أو إعدادها من جديد على كل جهاز، أو استخدام صورة نظام كاملة.

| الخدمة | المسار / المنفذ | الاستخدام | إعادة التثبيت على جهاز جديد |
|--------|-----------------|-----------|-----------------------------|
| **Jellyfin** | تثبيت نظام (`apt install jellyfin` أو غيره) — المنفذ 8096 | خدمة "الاستراحة" الافتراضية | `sudo apt install jellyfin` (أو اتباع دليل التوزيعة) |
| **MistServer** | يعمل عادة على `localhost:4242` | البث والـ streaming | تثبيت/تشغيل MistServer حسب وثائقه الرسمية |
| **zero-network-helper** | وحدة systemd من `deploy/zero-network-helper.service` | إعدادات الشبكة (DHCP/Static) من لوحة التحكم | نسخ الوحدة إلى `/etc/systemd/system/` ثم `systemctl enable --now zero-network-helper` |
| **NetworkManager** | نظام | مطلوب لـ zero-network-helper | `sudo apt install network-manager` |

---

## 4. خيارات النشر على أجهزة أخرى

### أ) نسخ الملفات + إعادة البناء (بدون صورة نظام)

- **تنقل:** مجلد Zero كاملاً (بدون `node_modules` وبدون `__pycache__` وبدون `frontend/build` لخفض الحجم)، مع `database.db` و`key.json` والملفات المرفوعة.
- **على الجهاز الجديد:**  
  - تثبيت Python3 و Node.js و npm.  
  - تثبيت الخدمات النظامية التي تحتاجها (Jellyfin، MistServer، NetworkManager).  
  - تشغيل سكربت التثبيت (مثلاً `deploy/install-from-bundle.sh`) الذي يشغل `pip install` و `npm install` و `npm run build` لـ Zero وللمشاريع المرتبطة (القرآن، قافية).  
- **النتيجة:** لا تحتاج صورة نظام، لكنك **تعيد تثبيت كل المشاريع والخدمات يدوياً أو عبر السكربت**.

### ب) صورة نظام كاملة (سناب شوت مع لينكس)

- **تنقل:** صورة كاملة للقرص أو للجهاز (نظام التشغيل + Zero + مشاريعه + Jellyfin + MistServer + الإعدادات).
- **على الجهاز الجديد:** استعادة الصورة على جهاز بمواصفات مشابهة.
- **النتيجة:** لا إعادة تثبيت للمشاريع أو الخدمات؛ كل شيء كما كان. مناسب عندما يكون لديك عدة أجهزة متشابهة ولا تريد تكرار إعداد الخدمات والمشاريع المرتبطة.

---

## 5. خلاصة

- **إعادة البناء وحدها لا تشمل:** الخدمات النظامية (Jellyfin، MistServer) ولا تثبيت الاعتماديات لمشاريع القرآن وقافية إلا إذا نفذت أوامرها صراحة أو استخدمت سكربت التثبيت.
- **لتفادي إعادة تثبيت كل شيء يدوياً:** إمّا استخدام **صورة نظام كاملة**، أو استخدام **حزمة نشر + سكربت تثبيت** يوثق وينفذ تثبيت Zero ومشاريعه المرتبطة (القرآن، قافية) وتوثق خطوات تثبيت Jellyfin و MistServer يدوياً إن لزم.

لمزيد من التفاصيل انظر:
- `deploy/README.md` — خدمة zero-network-helper.
- `QUICK_START.md` — تشغيل المشروع ووضع الإنتاج.
- **حزمة النشر:** من جذر المشروع نفّذ `./deploy/make-bundle.sh` لإنشاء أرشفة `zero-deploy-YYYYMMDD-HHMM.tar.gz`. انقلها إلى الجهاز الجديد، استخرجها ثم نفّذ من داخل المجلد المستخرج: `./deploy/install-from-bundle.sh` (سيثبت Backend والواجهة والقرآن وقافية).
