# إصلاحات بعد نسخ القرص (سناب شوت) إلى جهاز جديد

عند نسخ قرص كامل من جهاز إلى جهاز آخر، يبقى النظام يعتقد أنه الجهاز القديم. هذا الملف يوضح **ما يتحتم إصلاحه** لتجنب التعارضات والأخطاء، وخطوات التطبيق.

---

## 1. هوية الجهاز (يجب تغييرها على الجهاز الجديد)

| البند | السبب | الإجراء |
|------|--------|---------|
| **machine-id** | يستخدمه systemd و D-Bus؛ إذا بقي كما في الجهاز القديم قد يحدث تعارض في الشبكة أو الخدمات. | إعادة توليده (انظر السكربت أو الأوامر أدناه). |
| **hostname** | اسم الجهاز يظهر في الشبكة والسجلات؛ جهازان بنفس الاسم يسببان التباساً. | تعيين اسم فريد للجهاز الجديد. |
| **D-Bus machine-id** | نسخة أخرى من هوية الجهاز. | تُحدَّث تلقائياً مع machine-id إن استخدمت السكربت. |

**ملاحظة:** تطبيق Zero يقرأ الرقم التسلسلي و UUID من `/sys/class/dmi/id` عند كل تشغيل، لذلك **لوحة التحكم ستعرض تلقائياً هوية الجهاز الجديد** بعد النسخ ولا تحتاج تعديلاً من داخل التطبيق.

---

## 2. مفاتيح SSH للخادوم (يجب تغييرها)

| البند | السبب | الإجراء |
|------|--------|---------|
| **مفاتيح SSH (host keys)** | كل خادوم يجب أن يكون له مفاتيح فريدة؛ نسخ نفس المفاتيح يضعف الأمان ويُربك العملاء. | حذف المفاتيح القديمة وتوليد مفاتيح جديدة. |

---

## 3. الشبكة والعنونة

| البند | السبب | الإجراء |
|------|--------|---------|
| **عنوان IP الثابت** | إذا كان الجهاز القديم يستخدم IP ثابتاً، قد يتعارض مع أجهزة أخرى على الشبكة. | على الجهاز الجديد: إما استخدام DHCP أو تعيين IP ثابت جديد مختلف. |
| **إعدادات المحول (interface)** | أسماء الواجهات (مثل eth0, enp0s3) قد تختلف بين الأجهزة. | التحقق من أسماء الواجهات وإعدادات Netplan/NetworkManager حسب الجهاز الجديد. |
| **قواعد 70-persistent-net.rules (udev)** | قد تربط أسماء واجهات بـ MAC الجهاز القديم. | على النظامات الحديثة غالباً لا تُستخدم؛ إن وُجدت وسبّبت تعارضاً يمكن إزالتها أو تعديلها. |

---

## 4. الأقراص وملف fstab

| البند | السبب | الإجراء |
|------|--------|---------|
| **UUID الأقسام** | بعد النسخ، أقسام القرص على الجهاز الجديد لها UUID مختلفة. | التحقق من `lsblk -f` وملف `/etc/fstab`؛ استبدال UUID القديمة بـ UUID الحالية إن لزم. |
| **تسميات الأقراص** | أسماء مثل /dev/sda قد تختلف (مثلاً /dev/sdb). | الاعتماد على UUID أو PARTLABEL في fstab أفضل من أسماء الأجهزة. |

---

## 5. التحديثات والتعريفات

| البند | السبب | الإجراء |
|------|--------|---------|
| **حزم النظام** | نواة وتعريفات الجهاز القديم قد لا تناسب الجهاز الجديد. | تشغيل `apt update && apt upgrade` (أو ما يعادله في التوزيعة). |
| **النواة (Kernel)** | قد تحتاج نواة أو وحدات مناسبة لهاردوير الجهاز الجديد. | بعد التحديث إعادة التشغيل؛ إن لزم تثبيت حزم نواة إضافية (مثل linux-image-generic). |
| **تعريفات الهاردوير** | رسوميات، شبكة، تخزين قد تختلف. | عادة تُحمَّل تلقائياً بعد التحديث وإعادة التشغيل؛ إن ظهرت مشاكل راجع تعريفات التوزيعة أو الشركة الصانعة. |

---

## 6. أشياء أخرى قد تتعارض

| البند | السبب | الإجراء |
|------|--------|---------|
| **رخص أو تفعيل مرتبط بالجهاز** | بعض البرامج تربط الترخيص بـ machine-id أو الهاردوير. | بعد تغيير machine-id قد تحتاج إعادة تفعيل؛ راجع دليل البرنامج. |
| **شهادات SSL مرتبطة بالاسم** | إن كانت الشهادة مرتبطة بـ hostname الجهاز القديم. | بعد تغيير hostname قد تحتاج تجديد الشهادة أو استخدام الاسم الجديد. |
| **قواعد جدار النار أو قوائم سماح** | قواعد مرتبطة بـ IP أو hostname الجهاز القديم. | مراجعة UFW/iptables وأي قوائم سماح (مثلاً في خواديم بعيدة) وتحديثها للجهاز الجديد. |
| **سجلات ومراقبة** | أنظمة مراقبة تعتمد على hostname أو IP. | تحديث إعدادات المراقبة لتعكس الجهاز الجديد. |

---

## 7. تطبيق الإصلاحات

### الطريقة الأولى: سكربت تلقائي (موصى به للخطوات الآمنة)

من مجلد المشروع (بعد نسخ القرص واستعادة النظام على الجهاز الجديد):

```bash
sudo ./deploy/post-clone-fixes.sh
```

السكربت سيقوم بـ:
- إعادة توليد `machine-id` وربط D-Bus به.
- طلب إدخال **hostname** جديد وتعيينه.
- حذف مفاتيح SSH القديمة وتوليد مفاتيح جديدة.
- تشغيل `apt update && apt upgrade -y`.

باقي الخطوات (مثل ضبط IP الثابت أو fstab) تبقى يدوية حسب حاجتك؛ راجع الأقسام أعلاه.

### الطريقة الثانية: أوامر يدوية

```bash
# 1) إعادة توليد machine-id
sudo rm -f /etc/machine-id /var/lib/dbus/machine-id
sudo systemd-machine-id-setup

# 2) تعيين hostname جديد (استبدل NEW-HOSTNAME بالاسم المطلوب)
sudo hostnamectl set-hostname NEW-HOSTNAME
# إن استخدمت /etc/hosts:
echo "127.0.1.1 NEW-HOSTNAME" | sudo tee -a /etc/hosts

# 3) مفاتيح SSH جديدة
sudo rm -f /etc/ssh/ssh_host_*
sudo dpkg-reconfigure openssh-server

# 4) التحديثات
sudo apt update && sudo apt upgrade -y

# 5) إعادة التشغيل
sudo reboot
```

بعد إعادة التشغيل تحقق من:
- `hostname`
- `cat /etc/machine-id`
- عنوان IP والشبكة (مثلاً `ip a` أو إعدادات Netplan/NetworkManager).
- إن كان لديك أقسام إضافية أو fstab مخصص: `lsblk -f` و `cat /etc/fstab`.

---

## 8. ملخص سريع

| ما يُصلح تلقائياً (سكربت) | ما تتحقق منه يدوياً بعد النسخ |
|---------------------------|----------------------------------|
| machine-id و D-Bus         | عنوان IP (ثابت أو DHCP)         |
| hostname                  | أسماء الواجهات و Netplan        |
| مفاتيح SSH للخادوم        | ملف fstab و UUID الأقسام         |
| تحديث الحزم (apt)         | جدار النار وقوائم السماح        |
| —                         | أي رخص أو شهادات مرتبطة بالجهاز |

بعد تطبيق هذه الإصلاحات، هوية الجهاز في النظام وفي تطبيق Zero (الرقم التسلسلي و UUID في لوحة التحكم) ستطابق الجهاز الجديد، وتقل التعارضات مع الجهاز المصدر أو مع أجهزة أخرى على الشبكة.
