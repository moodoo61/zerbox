# دليل تنزيل وتثبيت المشروع على أجهزة أخرى

هذا الدليل يشرح كيف تنزّل مشروع **zerbox** من GitHub وتثبّته على جهاز جديد.  
**مسار البناء والتثبيت في هذا الدليل:** `~/zerbox`

---

## 1. المتطلبات قبل البدء

يجب أن يكون على الجهاز:

| المطلوب | الاستخدام | التحقق |
|---------|-----------|--------|
| **Git** | تنزيل المشروع من GitHub | `git --version` |
| **Python 3** | تشغيل الـ Backend | `python3 --version` |
| **pip** | تثبيت مكتبات Python | `pip3 --version` |
| **Node.js** و **npm** | بناء الواجهة الأمامية | `node --version` و `npm --version` |

### تثبيت المتطلبات (Ubuntu / Debian)

```bash
sudo apt update
sudo apt install -y git python3 python3-pip nodejs npm
```

إذا كان Node.js قديماً أو غير موجود، يمكن تثبيته من الموقع الرسمي: https://nodejs.org

---

## 2. تنزيل المشروع إلى `~/zerbox`

تنزيل المستودع داخل مجلد المنزل باسم **zerbox**:

```bash
cd ~
git clone https://github.com/moodoo61/zerbox.git
cd zerbox
```

بعدها يكون المشروع في **`~/zerbox`** (أي `$HOME/zerbox`).

---

## 3. تثبيت التبعيات والتشغيل

لديك خياران: **سكربت تلقائي** (موصى به) أو **خطوات يدوية**.

---

### الطريقة أ: سكربت تثبيت واحد (موصى به)

من مجلد المشروع **`~/zerbox`**:

```bash
cd ~/zerbox
./deploy/install-from-bundle.sh
```

السكربت يقوم بـ:
1. تثبيت اعتماديات الـ Backend (Python)
2. تثبيت وبناء الواجهة الأمامية (React)
3. تثبيت اعتماديات قرآن وقافية إن وُجدت

بعد انتهاء السكربت بدون أخطاء، انتقل إلى **القسم 4** للتشغيل.

---

### الطريقة ب: خطوات يدوية

نفّذ من **`~/zerbox`**:

```bash
cd ~/zerbox

# 1) Backend
pip3 install -r backend/requirements.txt

# 2) Frontend — تثبيت ثم بناء
cd frontend
npm install
npm run build
cd ..

# 3) (اختياري) قرآن
# إذا وُجد frontend/quran:
#   cd frontend/quran && npm install && cd ../..

# 4) (اختياري) قافية
# إذا وُجد qafiyah وله package.json:
#   cd qafiyah && npm install && cd ..
```

بعدها انتقل إلى **القسم 4**.

---

## 4. تشغيل المشروع

من **`~/zerbox`**:

```bash
cd ~/zerbox
./start-production.sh
```

- الواجهة والـ API ستكون على المنفذ **8000**
- افتح المتصفح: **http://localhost:8000**  
  أو من جهاز آخر على نفس الشبكة: **http://عنوان-الجهاز:8000**

لإيقاف التشغيل: `Ctrl+C` في نفس الطرفية.

---

## 5. (اختياري) التشغيل التلقائي عند إقلاع الجهاز

إذا كان الجهاز Linux ويستخدم **systemd**، يمكن جعل zerbox يعمل كخدمة. السكربت يضبط المسار تلقائياً على **`~/zerbox`**:

```bash
cd ~/zerbox
ZERO_ROOT=~/zerbox ./deploy/install-services.sh

sudo systemctl enable zero
sudo systemctl start zero
sudo systemctl status zero
```

لإضافة دعم الشبكة والهوتسبوت (ZeroLAG):

```bash
sudo apt install -y network-manager
sudo systemctl enable zero-network-helper
sudo systemctl start zero-network-helper
```

تفاصيل أكثر في: [deploy/README.md](../deploy/README.md).

---

## 6. إذا ثبّت المشروع في مسار غير `~/zerbox`

إذا استنسخت المشروع في مسار آخر (مثل `/opt/zerbox`)، استبدل **`~/zerbox`** بذلك المسار في كل الأوامر أعلاه. لخدمات systemd استخدم:

```bash
ZERO_ROOT=/المسار/الفعلي/zerbox ./deploy/install-services.sh
```

---

## 7. ملفات لا تأتي مع المستودع (اختياري)

هذه الملفات **لا تُرفع** إلى GitHub لأسباب أمنية أو لأنها تُنشأ محلياً:

| الملف | الوصف |
|-------|--------|
| `database.db` / `app.db` | قاعدة البيانات — تُنشأ تلقائياً عند أول تشغيل |
| `.env` | متغيرات بيئة (إن احتجتها) |
| `key.json` / `kay.json` | مفاتيح البث (إن استخدمت ميزة البث) |
| `vapid_keys.json` | مفاتيح Web Push للإشعارات |

لا تحتاج إنشاء شيء للتشغيل الأساسي؛ إن احتجت البث أو الإشعارات أضف الملفات المناسبة في جذر المشروع حسب توثيق المشروع.

---

## 8. ملخص سريع (نسخ ولصق)

على جهاز جديد (بعد تثبيت Git و Python و Node/npm):

```bash
cd ~
git clone https://github.com/moodoo61/zerbox.git
cd zerbox

./deploy/install-from-bundle.sh
./start-production.sh
```

ثم افتح المتصفح على: **http://localhost:8000**

---

## 9. التحديث من GitHub لاحقاً

عندما تريد جلب آخر التعديلات من المستودع:

```bash
cd ~/zerbox
git pull
# إعادة بناء الواجهة إن تغيّر frontend
cd frontend && npm install && npm run build && cd ..
./start-production.sh
```

إذا استخدمت systemd:
```bash
sudo systemctl restart zero
```
