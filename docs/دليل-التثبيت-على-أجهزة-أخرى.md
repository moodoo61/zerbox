# دليل تنزيل وتثبيت المشروع على أجهزة أخرى

هذا الدليل يشرح كيف تنزّل مشروع **zerbox** من GitHub وتثبّته على جهاز جديد بشكل كامل.  
**مسار التثبيت الافتراضي:** `/root/zerbox`

---

## 1. متطلبات النظام

| المطلوب | الاستخدام | الحد الأدنى |
|---------|-----------|-------------|
| **Ubuntu / Debian** | نظام التشغيل | Ubuntu 22.04+ |
| **Python 3** | Backend (FastAPI) | 3.10+ |
| **Node.js** | بناء الواجهة الأمامية والخدمات | **v22.x** |
| **npm** | إدارة حزم JavaScript | 10.x+ |
| **pnpm** | تشغيل مشروع قافية (monorepo) | 9.x+ |
| **Git** | تنزيل المشروع | أي إصدار |
| **MistServer** | خادم البث (RTMP/HLS) | مطلوب للبث |
| **xl2tpd + ppp** | اتصال L2TP VPN | مطلوب |
| **Docker + Docker Compose** | قاعدة بيانات قافية محلية (اختياري) | اختياري |

---

## 2. تثبيت المتطلبات الأساسية

### 2.1 تحديث النظام وتثبيت الأدوات الأساسية

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git python3 python3-pip curl wget build-essential
```

### 2.2 تثبيت Node.js v22.x (الإصدار المطابق للمشروع)

> **مهم:** إصدار Node.js من مستودعات Ubuntu الافتراضية قديم جداً. يجب تثبيت v22.x.

```bash
# تثبيت Node.js v22.x عبر NodeSource
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# التحقق من الإصدار
node --version   # يجب أن يظهر v22.x.x
npm --version    # يجب أن يظهر 10.x.x
```

### 2.3 تثبيت pnpm (مطلوب لمشروع قافية)

```bash
npm install -g pnpm@9
pnpm --version   # يجب أن يظهر 9.x.x
```

### 2.4 تثبيت حزم VPN والصوت

```bash
sudo apt install -y xl2tpd ppp beep
```

### 2.5 تثبيت MistServer (خادم البث)

```bash
# تنزيل وتثبيت MistServer
curl -o - https://releases.mistserver.org/is/mistserver_64V3.10.tar.gz 2>/dev/null | sh
```

> **ملاحظة:** واجهة إدارة MistServer تعمل على المنفذ `4242`  
> بيانات الدخول الافتراضية: `admin` / `admin`

### 2.6 تثبيت Jellyfin (اختياري — خدمة الاستراحة)

```bash
# إضافة مستودع Jellyfin
curl -fsSL https://repo.jellyfin.org/install-debuntu.sh | sudo bash

# تفعيل الخدمة
sudo systemctl enable jellyfin
sudo systemctl start jellyfin
```

> واجهة Jellyfin تعمل على المنفذ `8096`

## 3. تنزيل المشروع

### 3.1 تنزيل zerbox

```bash
cd /root
git clone https://github.com/moodoo61/zerbox.git
cd zerbox
```

### 3.2 تنزيل قافية والقرآن الكريم

إذا لم تكن مجلدات **قافية** أو **القرآن الكريم** موجودة داخل المشروع، انسخ الأوامر التالية ولصقها — لا حاجة للبحث في مكان آخر:

**تنزيل قافية (منصة الشعر العربي) داخل zerbox:**

```bash
cd /root/zerbox
git clone https://github.com/alwalxed/qafiyah.git qafiyah
```

**تنزيل تطبيق القرآن الكريم (Vue) داخل zerbox:**

```bash
cd /root/zerbox
git clone https://github.com/harran55/vuejs-quran-web.git quran
```

> **ملاحظة:** إذا كان المستودع zerbox يحتوي بالفعل على `qafiyah/` أو `quran/` يمكن تخطي الأمر المناسب. إذا نفّذت `git clone` فوق مجلد موجود سيفشل الأمر — احذف المجلد أولاً أو استخدم مساراً آخر ثم انقل المحتوى.

بعد التنزيل يصبح الهيكل كالتالي:
- `zerbox/qafiyah/` — مشروع قافية (monorepo)
- `zerbox/quran/` — تطبيق القرآن الكريم

## 4. تثبيت التبعيات والبناء

### الطريقة أ: سكربت تلقائي شامل (موصى به)

```bash
cd /root/zerbox
chmod +x deploy/install-from-bundle.sh
./deploy/install-from-bundle.sh
```

السكربت يقوم بـ:
1. تثبيت اعتماديات Python (Backend)
2. تثبيت وبناء الواجهة الأمامية (React)
3. تثبيت وبناء تطبيق القرآن الكريم (Vue)
4. تثبيت وبناء مشروع قافية (Next.js عبر pnpm + turbo)

---

### الطريقة ب: خطوات يدوية

```bash
cd /root/zerbox

# 1) Backend — مكتبات Python
pip3 install -r backend/requirements.txt

# 2) الواجهة الأمامية الرئيسية — React
cd frontend
npm install
npm run build
cd ..

# 3) القرآن الكريم — Vue 2 (مصدر: vuejs-quran-web)
cd quran
npm install
npm run build
cd ..

# 4) قافية — Next.js (pnpm monorepo، مصدر: alwalxed/qafiyah)
cd qafiyah
pnpm install
pnpm build
cd ..
# اختياري: قاعدة بيانات محلية — يتطلب Docker ثم: ./scripts/setup-database.sh
```

## 5. إعداد القرآن الكريم وقافية

يُدرج أدناه إعداد **القرآن الكريم** (Vue) و**قافية** (منصة الشعر العربي) مع روابط المستودعات الأصلية والخطوات الكاملة للتطوير والإنتاج.

### 5.1 القرآن الكريم

تطبيق الويب **القرآن الكريم** مبني على [vuejs-quran-web](https://github.com/harran55/vuejs-quran-web) (Vue 2 + Vue CLI).

- **المصدر الأصلي:** https://github.com/harran55/vuejs-quran-web  
- **المسار في zerbox:** `quran/`  
- **المنفذ:** `8081`  
- **أمر التطوير:** `npm run serve -- --port 8081`  
- **أمر البناء للإنتاج:** `npm run build` (ينشئ مجلد `dist/`)

**التشغيل في وضع التطوير (يدوياً):**

```bash
cd /root/zerbox/quran
npm install
npm run serve -- --port 8081
```

**البناء للإنتاج (اختياري — لخدمة الملفات الثابتة):**

```bash
cd /root/zerbox/quran
npm run build
# المخرجات في dist/ — يمكن تقديمها بأي خادم ثابت (مثلاً nginx)
```

**تعديل المسار الأساسي للتطبيق (إذا نُشر في مسار فرعي):**  
عدّل في المشروع وسم `<meta name="url" content="/المسار/">` حسب المسار المطلوب.

التطبيق يُدار من لوحة التحكم (قسم **الخدمات الافتراضية**). يمكن تفعيله/تعطيله من الإعدادات.

---

### 5.2 قافية (منصة الشعر العربي)

مشروع **قافية** مبني على [Qafiyah](https://github.com/alwalxed/qafiyah) — منصة مفتوحة المصدر للشعر العربي (أكثر من 944 ألف بيت، 932 شاعراً).

- **المصدر الأصلي:** https://github.com/alwalxed/qafiyah  
- **المسار في zerbox:** `qafiyah/` (monorepo)  
- **الواجهة الأمامية:** `qafiyah/apps/web` (Next.js) — المنفذ `8082`  
- **المتطلبات حسب المستودع:** Node.js 18+، pnpm 8+، واختيارياً **Docker** لقاعدة البيانات المحلية.

**محتوى الـ monorepo:**
- `apps/web` — واجهة Next.js (المنفذ `8082`)
- `apps/api` — خادم Hono REST API
- `apps/bot` — بوت تويتر
- `packages/schemas` — مخططات Zod المشتركة

**التشغيل بدون قاعدة بيانات محلية (يعتمد على الـ API العام):**

```bash
cd /root/zerbox/qafiyah
pnpm install
cd apps/web
npm run dev -- -p 8082
```

**التشغيل مع قاعدة بيانات محلية (اختياري — للبحث والبيانات الكاملة):**

1. تثبيت Docker و Docker Compose.
2. من جذر قافية تشغيل سكربت إعداد القاعدة:

```bash
cd /root/zerbox/qafiyah
./scripts/setup-database.sh
```

السكربت ينشئ حاويتي PostgreSQL (منفذ التطوير `5433`، منفذ الاختبار `5434`) ويكتب ملفات `.dev.vars` تلقائياً. بعدها:

```bash
pnpm dev
```

التطبيق سيعمل على `http://localhost:3000` (أو استخدم `-p 8082` من `apps/web` إذا شغّلته من هناك).

**للإنتاج (خادم حقيقي):** ضبط `DATABASE_URL` في `.env.prod` حسب قاعدة البيانات المستخدمة، ثم من جذر قافية:

```bash
pnpm build
```

**تشغيل يدوي من zerbox (المنفذ 8082):**

```bash
cd /root/zerbox/qafiyah/apps/web
npm run dev -- -p 8082
```

قافية تُدار من لوحة التحكم عبر قسم **الخدمات الافتراضية**. يمكن تفعيلها/تعطيلها من الإعدادات.

## 6. تثبيت الخدمات وتشغيلها تلقائياً

### 6.1 تثبيت خدمات systemd

```bash
cd /root/zerbox
chmod +x deploy/install-services.sh
./deploy/install-services.sh
```

### 6.2 تفعيل التشغيل التلقائي لجميع الخدمات

```bash
# الخدمة الرئيسية (Backend + Frontend)
sudo systemctl enable zero
sudo systemctl start zero

# خدمة الشبكة (الهوتسبوت والإعدادات)
sudo systemctl enable zero-network-helper
sudo systemctl start zero-network-helper

# MistServer (خادم البث)
sudo systemctl enable mistserver
sudo systemctl start mistserver

# xl2tpd (خدمة VPN — يتصل تلقائياً من داخل المشروع)
sudo systemctl enable xl2tpd
sudo systemctl start xl2tpd
```

### 6.3 التحقق من حالة الخدمات

```bash
sudo systemctl status zero
sudo systemctl status zero-network-helper
sudo systemctl status mistserver
sudo systemctl status xl2tpd
```

### 6.4 أوامر مفيدة لإدارة الخدمات

```bash
# إعادة تشغيل الخدمة الرئيسية
sudo systemctl restart zero

# عرض سجلات الخدمة
sudo journalctl -u zero -f

# إيقاف الخدمة
sudo systemctl stop zero
```

## 7. التشغيل اليدوي (بدون systemd)

إذا أردت التشغيل يدوياً بدلاً من الخدمات:

```bash
cd /root/zerbox
./start-production.sh
```

- الواجهة والـ API: **http://عنوان-الجهاز:8000**
- لوحة التحكم: **http://عنوان-الجهاز:8000/admin**
- بيانات الدخول الافتراضية: `admin` / `admin`

لإيقاف التشغيل: `Ctrl+C`

## 13. استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| `node: command not found` | تأكد من تثبيت Node.js v22.x عبر NodeSource (القسم 2.2) |
| `pnpm: command not found` | `npm install -g pnpm@9` |
| VPN لا يتصل | `sudo apt install xl2tpd ppp` ثم `sudo systemctl restart xl2tpd` |
| MistServer لا يعمل | `sudo systemctl restart mistserver` |
| الواجهة لا تظهر | `cd frontend && npm run build` ثم أعد تشغيل الخدمة |
| خطأ في Python | `pip3 install -r backend/requirements.txt` |
| القرآن: `vue-cli-service: command not found` | من `quran`: `npm install` ثم `npm run serve -- --port 8081` |
| القرآن: صفحة بيضاء أو أخطاء في المسارات | تحقق من وسم `<meta name="url" content="...">` في `quran/` إن كان التطبيق في مسار فرعي |
| قافية: `pnpm: command not found` | `npm install -g pnpm@9` (أو pnpm 8+ حسب المستودع) |
| قافية: فشل `pnpm build` | تأكد من Node.js 18+؛ جرّب من الجذر: `cd qafiyah && pnpm install && pnpm build` |
| قافية: إعداد قاعدة البيانات فشل | تأكد من تثبيت Docker و Docker Compose وتشغيل `./scripts/setup-database.sh` من جذر `qafiyah` |
