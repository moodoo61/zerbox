# دليل: ماذا تفعل بعد كل تعديل

بعد أي تغيير في المشروع (كود، إعدادات، واجهة، إلخ) اتبع الخطوات التالية لرفع التعديلات إلى GitHub.

---

## الخطوات من الطرفية (Terminal)

افتح الطرفية وتأكد أنك في مجلد المشروع، ثم نفّذ الأوامر بالترتيب.

### 1) الانتقال لمجلد المشروع

```bash
cd /root/Zero
```

### 2) عرض ما تغيّر

```bash
git status
```

يعرض الملفات المعدّلة أو الجديدة. راجع القائمة وتأكد أنك لا ترفع ملفات حساسة (مثل `.env` أو قواعد بيانات).

### 3) إضافة التعديلات

**إضافة كل الملفات المعدّلة:**
```bash
git add .
```

**أو إضافة ملفات محددة فقط:**
```bash
git add backend/main.py frontend/src/App.js
```

### 4) عمل commit مع وصف واضح

```bash
git commit -m "وصف التعديل بالعربي أو الإنجليزي"
```

أمثلة:
```bash
git commit -m "إصلاح تسجيل الدخول"
git commit -m "إضافة صفحة الإعدادات"
git commit -m "Fix streaming API timeout"
```

### 5) رفع التعديلات إلى GitHub

```bash
git push
```

إذا كان الفرع الأول مرة أو ظهرت رسالة تطلب ربط الفرع:
```bash
git push -u origin main
```

---

## ملخص سريع (نسخ ولصق)

بعد أي تعديل، من مجلد `/root/Zero`:

```bash
cd /root/Zero
git add .
git commit -m "وصف التعديل هنا"
git push
```

---

## ماذا لو نسيْتَ التعديلات ولم تعمل commit؟

- التعديلات تبقى على جهازك فقط حتى تنفّذ `git add` و `git commit` ثم `git push`.
- يمكنك في أي وقت تشغيل `git status` لرؤية الملفات التي لم تُرفع بعد.

---

## ماذا لو أردتَ التراجع عن تعديل في ملف (قبل الـ commit)؟

- إرجاع ملف واحد لحالة آخر commit:
  ```bash
  git checkout -- path/to/file
  ```
- إلغاء إضافة ملف من منطقة الـ staging (بعد `git add` وقبل `git commit`):
  ```bash
  git reset HEAD path/to/file
  ```

---

## ملفات لا تُرفع (موجودة في .gitignore)

لا تُضاف تلقائياً ولن تُرفع إلى GitHub، ومنها:

- `node_modules/` و `__pycache__/`
- `.env` و `database.db` و `app.db`
- `key.json` و `kay.json` و `vapid_keys.json`
- `frontend/build/`

إذا ظهرت في `git status` وتريد استثناءها، أضف مسارها إلى `.gitignore` في جذر المشروع.
